## 1. 背景

搜索链路依赖 BE 全文检索契约，契约未冻结前不应进入当前候选版本。

## 2. 目标

- 明确 FE-013 搜索链路的冻结边界。
- 在 BE 全文检索契约冻结前，禁止搜索链路进入 Ready。
- 将 smoke 与回滚口径前置为进入候选版本的必要条件。

## 3. 范围
- In Scope:
  - FE-013
- Out of Scope:
  - 并入当前候选版本
  - FE-014 及更后续链路

## 4. 契约
- API / 数据结构 / 错误语义
  - 搜索相关契约未冻结前，不允许通过实现分支改变搜索入口、字段或错误语义
  - 若搜索契约需要调整，必须先完成新的 OpenSpec change

## 5. 验收标准
- [ ] 冻结边界明确且可追踪
- [ ] Harness 计划包含构建、关键 smoke 与回滚证据
- [ ] 搜索链路在契约未冻结前不会进入 Ready
- [ ] 延后项与回滚路径在低权重同步记录中可复核

## 6. 风险与回滚
- 风险:
  - 搜索契约未冻结时若误推进，会造成候选版本范围漂移
  - 回滚口径不明确会导致搜索链路难以安全延期
- 回滚:
  - 继续保持 FE-013 延后，不并入当前候选版本
  - 若已误进入候选版本，退回上一冻结 commit / release tag
