## Overview

本次变更的目的不是推动搜索链路立即进入实现，而是先把它从候选版本中隔离出来，直到 BE 全文检索契约、smoke 计划和回滚动作都已经对齐。

## Design Decisions

### 1. 冻结边界先于实现

- FE-013 搜索链路不进入当前候选版本的 Ready 状态。
- 是否进入实现分支，取决于契约冻结和 Harness 前置是否完成。

### 2. 短期状态外置

- 当前候选版状态、风险台账、延期说明放入低权重同步记录。
- 稳定规则正文只保留长期有效的边界与门禁要求。

### 3. Harness 计划先行

- 先定义搜索链路的 smoke 与回滚口径，再允许其进入实现或候选版。
- 若缺少可执行验证计划，则继续保持冻结。

## Artifact Mapping

- `proposal.md`: 说明为什么搜索链路需要延期与冻结。
- `specs/fe-search-freeze-boundary/spec.md`: 定义冻结边界、延期准则与验证要求。
- `tasks.md`: 定义同步记录、验证口径与回写动作。

## Failure / Rollback Path

- 若 BE 全文检索契约未冻结，则继续保持延期，不追加到候选版本。
- 若冻结边界需要调整，必须通过新的 OpenSpec change 重新定义，不得在实现层直接松绑。
