# MVP Scope

## 目标

冻结 Phase 1 为“邮箱验证码注册/登录 + JWT 鉴权 + 文章读写持久化 + 游客浏览”的 MVP 最小内容闭环。

## 必须完成

### FE

- 登录/注册流程（邮箱验证码 + 密码）
- 受保护写操作的 JWT 鉴权接入
- 管理端文章创建/编辑/删除/发布
- 游客可访问的已发布文章列表
- 游客可访问的文章详情
- Phase 1 核心链路不依赖 mock 数据

### BE

- 登录/注册流程（邮箱验证码 + 密码）
- JWT 鉴权（含受保护接口访问控制）
- 文章创建/编辑/删除/发布
- 已发布文章列表和详情读取
- 文章数据持久化到 PostgreSQL

### 数据与契约

- 文章写入 PostgreSQL 后可被前端再次读取
- 认证与文章相关 API 行为与契约逐步对齐

## 本次不做

- 评论、回复、点赞
- TinyMCE 富文本编辑器
- 图片上传、MinIO、OSS
- 全文搜索
- reCAPTCHA
- 复杂个人中心（头像上传、资料编辑、密码修改等）
- 复杂后台统计/运营面板
- 完整 CI/CD
- E2E 自动化测试
- 监控告警

## 完成判定

- 用户可完成邮箱验证码注册与登录，并通过 JWT 访问受保护入口
- 登录用户可完成文章创建、编辑、删除、发布
- 游客可浏览已发布文章列表与详情
- 文章读写链路完成 PostgreSQL 持久化并能被前端读取
- Phase 1 主流程验收不依赖评论、富文本、图片上传、搜索或平台级工程化能力
