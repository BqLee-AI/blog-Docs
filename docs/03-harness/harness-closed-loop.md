# Harness 闭环规范

## 核心定义

Harness 不等于“会跑命令”，而是让 Agent 在可控循环中执行：

执行 -> 观察 -> 验证 -> 收敛 -> 回写

## 四层能力

1. 执行环境层
   - 隔离工作目录
   - 可恢复运行环境

2. 运行时可见层
   - 构建日志
   - 接口日志
   - 前端页面行为（必要时截图）

3. 机械验证层
   - 类型检查 / 编译
   - 单元测试
   - 冒烟链路

4. 收敛层
   - 失败分类
   - 回滚策略
   - 经验回写

## FE 最低闭环

- `bun run build` 通过
- 关键页面可访问
- 认证链路（登录/注册/鉴权）至少一条 smoke

## BE 最低闭环

- `go test ./...` 通过
- 核心接口可调用
- 错误语义和状态码与约定一致

## 合并门槛

没有验证证据（命令输出摘要/截图/日志结论）的 PR 禁止合并。
