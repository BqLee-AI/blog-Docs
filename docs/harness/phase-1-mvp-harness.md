# Phase 1 MVP Harness

## 适用范围

- 本文档仅用于 Phase 1（MVP 最小内容闭环）验收。
- 本阶段验收范围限定为“账号认证 + 文章读写”。

## FE 验收

最小必跑检查：

- `bun run build`
- `bun run test`

判定规则：

- 两项命令均通过，且结果可复核，方可进入合并评审。
- 若存在 mock/mixed 数据依赖，不能作为 Phase 1 完成证据。

## BE 验收

最小必跑检查：

- `go test ./...`
- `go vet ./...`
- `go build ./...`

判定规则：

- 三项命令均通过，且结果可复核，方可进入合并评审。
- 涉及 API 行为变更时，需同步提供契约对齐信息。

## Contract 验收

- 契约真源为 `contracts/api/openapi.yaml`。
- 涉及认证或文章接口行为的任务，必须满足以下之一：
  - 已在 `contracts/api/openapi.yaml` 反映本次行为；
  - PR 中明确记录 OpenAPI gap，并关联后续契约任务。

最小覆盖域：

- Auth：验证码、注册、登录、当前用户、受保护入口。
- Articles：创建、列表、详情、编辑、删除、草稿/发布可见性。

## Smoke 验收

### 认证 smoke

- 验证码：可获取验证码（或等价测试口径）。
- 注册：可使用验证码完成注册。
- 登录：可成功登录并获得鉴权凭证。
- 当前用户：可读取当前登录用户信息。
- 受保护入口：未登录拒绝、已登录可访问。

### 文章 smoke

- 创建：登录用户可创建文章。
- 列表：可获取文章列表。
- 详情：可查看文章详情。
- 编辑：作者可编辑文章。
- 删除：作者可删除文章。
- 草稿/发布可见性：草稿与已发布在访客/作者视角的可见性符合约定。

## Evidence 要求

每项验收至少提供以下证据：

- Command/入口
- Result（PASS/FAIL/N/A + 原因）
- Evidence（日志片段、截图或报告链接）
- Conclusion（是否满足 Phase 1 门禁）

建议按 FE / BE / Contract / Smoke 分组提交证据，便于评审复核。

## 非 Phase 1 验收项

以下能力明确不作为 Phase 1 完成判定：

- 评论
- 富文本编辑
- 图片上传
- 全文搜索
- reCAPTCHA
