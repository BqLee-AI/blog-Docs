# AI Native 治理总则（blog-Docs）

## 目标

将 `blog-Docs` 建成前后端开发的控制面仓库，而不是资料仓库。

## 三层控制面

1. Prompt 控制：任务边界与验收口径
2. Context 控制：信息权威、新鲜度、加载范围
3. Harness 控制：执行、验证、回滚、审计闭环

## 治理原则

- 先定盘，后开发：Spec 和边界先于编码
- 先验证，后合并：没有验证证据不合并
- 先回写，后遗忘：每次踩坑必须沉淀为规则
- 先模块化，后扩张：规范短小、可复用、可演进

## 仓库职责边界

- 本仓库负责：规则、流程、模板、闭环机制
- FE/BE 仓库负责：业务实现与测试执行
- 本仓库不替代 FE/BE 的技术设计细节，但定义其交付约束

## 运行节奏

- 周节奏：周一定盘，周中执行，周末回写
- 里程碑：MVP 周期按 Week1~Week4 管控
- 长期：逐步从 Issue Driven 升级到 Spec + Test + AI-CICD
