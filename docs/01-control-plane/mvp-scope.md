# MVP Scope（范围冻结模板）

## 用途

这份文档不是记录某一周的项目状态，而是提供一份长期有效的范围冻结模板。

使用原则：

- 只写本次冻结范围必须保留的边界
- 不写容易快速过期的任务进度
- 不写具体负责人、具体 issue 编号、具体日期
- 一旦范围变化，应更新冻结版内容，而不是在文档里堆历史注释

## 建议结构

### 目标

用 1 到 3 句话说明本次冻结范围要交付什么，不解释长期路线，不展开过程细节。

### 必须完成

按系统边界写，而不是按人写：

- FE：哪些用户链路必须成立
- BE：哪些接口或领域能力必须成立
- Control Plane：哪些规则、契约、验证口径必须成立

### 本次不做

只写明确排除项，例如：

- 不做的大功能扩展
- 不做的重构
- 不做的流程跃迁
- 不采用的发版方式

### 延后项

只列类别，不列具体排期。

### 完成判定

建议从下面几类中挑选：

- 关键链路稳定
- 候选版本可发布
- 风险台账清晰
- 契约冻结完成
- 验证闭环成立

## 写法要求

- 优先写“边界”，不要写“进度”
- 优先写“系统必须成立什么”，不要写“谁这周要做什么”
- 避免写具体 issue、PR、分支名
- 如果需要记录阶段性状态，请放到 dashboard 或同步记录，而不是放在这里
